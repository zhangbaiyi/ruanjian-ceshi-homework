销售管理系统（第三题）

在一销售管理系统中，其中的一个模块负责对公司员工进行年终考评，考评综合考虑职工在公司工作时间长短（计年）、每年请假的次数（不能超过20天，20天以上，包括20天，则免于年终考评）、员工在公司的级别（分1，2，3，4，5个级别）及本年度的销售总额，考评的结果最高以5分计，公司成立于2000年初，该软件的设计使用周期到2035年底，请回答下列问题：

(1) 用基本边界值的测试方法，一共有多少测试用例；最坏情况边界值有多少测试用例。

基本边界值：理论上4*4+1=17个

时间长短（0，1，18，34，35）、每年请假次数（0，1，10，19，20）、员工级别（1，2，3，4，5）、本年度的销售总额（0，1），共有4+5+4+2+1=16个。

最坏情况边界值：555*2=250个

(2) 根据健壮的边界值测试法，写出“工作时间长短”为非正常值情况下的测试用例。

（-1，10，3，1），（36，10，3，1）

边界值（第五题）

边界值测试可以适用于function(method) level，class level,，system level，分析电商平台系统测试时考虑的边界值情况。

  1. 购物流程

电商平台系统的购物流程中，可能会出现商品数量限额，或是购物车限额的情况。此时，针对用户选择放入购物车的商品数量和商家库存、商家限额数量等，可设计边界值测试。如商品数量为0、1、正常值、与库存相等、超过库存、与限额相等、超过限额等情况。

  1. 支付流程

电商平台系统的支付流程中，结算时往往会出现优惠券、满减等情况。最终计算得出的支付金额可能出现负数、0和正常值。可针对优惠后金额设计边界值测试，如支付金额-1.00、0.00、50.00等情况。

double commission_cal(double sales, double cash_arrival, int off_days){

        

        
    double commission_factor, commission;

        
    if (sales > 2000000 && off_days <= 10){

        
        if (cash_arrival >= sales * 0.6){

        
            commission_factor = 7;

        
        }else{

        
            commission_factor = -1;

        
        }

        
    }else{

        
        if (cash_arrival <= sales * 0.85){

        
            commission_factor = 6;

        
        }else{

        
            commission_factor = 5;

        
        }

        
    }

}
